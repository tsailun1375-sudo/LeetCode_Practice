# LeetCode 11. Container With Most Water - 思維演進筆記

## 📌 題目核心
給定一組高度陣列，選取兩條線作為容器左右邊界，求出能盛裝的最大水量。
- **計算邏輯**：面積 = `兩邊較短者的高度` × `兩邊的距離`

---

## 💡 解法一：暴力法 (Brute Force) - 初始嘗試

### 思維路徑
這是我最開始想到的直覺方法。既然要找最大值，那就把所有可能的組合都算一遍，再從中找出最大的。我們使用兩層迴圈：
1. 第一層迴圈固定左邊界。
2. 第二層迴圈遍歷所有右邊界。

### 效能分析
- **時間複雜度**：$O(n^2)$
  - 假設有 $n$ 條線，組合數量大約是 $n^2 / 2$。當 $n = 10^5$ 時，運算次數會破百億，導致系統超時。
- **結果**：**TLE (Time Limit Exceeded)**。

---

## 🚀 解法二：雙指標法 (Two Pointers) - 優化後

### 思維轉變
為了突破效能瓶頸，我意識到不需要檢查「所有」組合。如果能透過某種規律，每次排除掉「絕對不可能成為最大值」的選項，就能大幅加速。

**核心邏輯：**
1. 將兩個指標分別放在最左端與最右端（此時寬度最大）。
2. **關鍵決策**：每次只移動**較矮**的那一邊。
3. **為什麼？** - 容器的高度是被「矮邊」限制的。
   - 如果我們移動「長邊」，寬度縮小了，但限制高度的依然是那條「矮邊」，面積只會變小。
   - 只有移動「矮邊」，才有機會遇到更高的邊來補償寬度縮小的損失。

### 效能分析
- **時間複雜度**：$O(n)$
  - 只需要從兩頭向中間掃描一遍，總共只走過 $n$ 個元素。
- **空間複雜度**：$O(1)$
  - 僅使用常數級變數存儲指標與最大面積。
- **結果**：**Accepted**。相較於暴力法，效能提升了數萬倍。

---

## 📈 複雜度轉變總結

| 指標 | 暴力解法 (Brute Force) | 雙指標法 (Two Pointers) |
| :--- | :--- | :--- |
| **時間複雜度** | $O(n^2)$ | $O(n)$ |
| **運算次數 (n=10^5)** | ~10,000,000,000 次 | ~100,000 次 |
| **效能表現** | 極慢 (會導致超時) | 極快 (秒殺) |

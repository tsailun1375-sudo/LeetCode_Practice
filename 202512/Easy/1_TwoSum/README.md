# 1_TwoSum (兩數之和)

### 解題思路
本題採用 **暴力搜尋法 (Brute Force)**。
透過兩層迴圈遍歷陣列中的所有組合，計算其和是否等於 `target`。

### 複雜度分析
- **時間複雜度**: $O(n^2)$  
  因為使用了雙重迴圈，最差情況下需要比較 $n \times (n-1) / 2$ 次。
- **空間複雜度**: $O(1)$  
  除了回傳的結果外，沒有使用額外的資料結構來儲存數據。

### 學習筆記
這是我的第一題 LeetCode！雖然 $O(n^2)$ 的解法可以通過，但在討論區看到其他人使用了 `unordered_map` 將時間複雜度優化到了 $O(n)$。這讓我理解到，同樣的問題可以用不同的演算法來提升效能。
